# 《不完美的小店》PLAN / PRD

## 1. 项目概述

《不完美的小店》（A Place Like Me）是一款 2D 小店经营 / 修补模拟 / 叙事向轻策略游戏。玩家经营一家旧物修补小店，每天面对不同顾客、旧物和请求，在收入、能量、材料和真实度之间做选择。

游戏的重点不是把小店经营成最高效率的商业机器，而是在有限资源下保留小店的温度：有些物品被完美修好，有些痕迹被认真保留，有些旧物则被重新创造为新的样子。

### 1.1 基础定位

- **游戏类型**：模拟经营类游戏，核心机制为能量、真实度、拒绝与主动休息。
- **视角与表现**：2D / 平面表现。
- **目标平台**：PC 单机，优先 Windows，16:9 横屏，推荐分辨率 `1920x1080`。
- **目标用户**：18–30 岁，偏好温暖叙事、轻度模拟经营、空间布置和情绪共鸣体验的玩家。
- **非目标方向**：不以硬核经营、动作战斗或多人竞技为主要方向。

### 1.2 设计主题

修复不是抹去破损，而是重新理解它的意义。

游戏关注的不是把所有东西修得像新的一样，而是让玩家思考：裂纹、褪色、磨损是否也可以成为记忆和经历的一部分。玩家需要在“修好物品”“理解顾客”和“照顾自己”之间找到平衡。

## 2. 体验目标

- **经营压力**：玩家需要考虑收入、材料、能量和次日状态，不能无限接单。
- **温柔修补**：修补结果不只有“完美”，保留痕迹和创意再造也能形成价值。
- **边界感选择**：面对加急、砍价、免费修改、改变风格等请求，玩家可以接受也可以拒绝。
- **不完美气质**：小店会成长，但不应该变成完全标准化、失去个性的店。
- **轻叙事反馈**：顾客反馈、店铺评价、日记和朋友圈共同塑造小店的生活感。

### 2.1 情感目标

- **被允许不完美的安心感**：玩家不需要把所有物品修到完美无缺，而是学习理解和装饰痕迹。
- **从经营成长中获得治愈感**：小店成长不仅是货架和装饰增加，也包括空间变得更柔和、更有生命力。
- **从他人故事中看见自己**：顾客故事对应害怕失败、自我否定、完美主义、无法接受过去错误等状态。
- **从修好物品转向接纳痕迹**：裂纹、补丁、歪斜、褪色不是失败条件，而是完成体验的一部分。

## 3. 核心循环

基础循环：

1. 查看订单
2. 选择订单
3. 选择修补方式
4. 消耗材料与能量
5. 获得收入 / 口碑 / 真实度变化
6. 处理手机消息或店铺事务
7. 开启下一天

该循环需要在 MVP 阶段就成立。后续系统都应围绕这个循环扩展，而不是打断它。

### 3.1 白天经营循环

白天是玩家主要进行经营和修复的阶段。玩家进入小店后，与店内需求栏交互，查看当天可接取的修复订单。需求栏显示物品类型、损坏程度、所需材料、消耗能量、预计报酬和顾客备注。

白天流程：

1. 查看需求栏
2. 选择订单
3. 选择修补方式
4. 修补旧物
5. 交付顾客
6. 获得收入与即时反馈

白天阶段的重点是经营决策：今天接什么订单、用什么方式修、是否值得消耗更多能量来换取收入或口碑。

### 3.2 夜晚小屋结算循环

晚上是整理一天结果和进入下一天的阶段。玩家回到小屋，通过手机查看今日收入、材料消耗、能量变化、口碑变化和顾客评价，也可以查看日记与朋友圈。

夜晚流程：

1. 回到小屋
2. 查看手机
3. 查看店铺评价 / 日记 / 朋友圈
4. 购入新的材料
5. 今日结算
6. 休息
7. 开启新一天

夜晚阶段不增加复杂操作，重点是让玩家回顾当天选择带来的结果，并感受到修复方式、边界选择和经营策略正在影响小店与主角状态。

### 3.3 长期成长循环

长期成长围绕“经营积累 → 店铺升级 → 订单变化 → 结局反馈”展开。成长重点不是把小店无限做大，而是让小店更稳定、更符合玩家的经营方式。

长期循环：

1. 完成每日订单
2. 积累金币与口碑
3. 购买材料 / 支付开销
4. 解锁小店升级
5. 吸引不同类型顾客
6. 影响真实度与结局走向

## 4. 核心数值

### 金币 / 收入

金币是小店维持运转的基础资源。玩家通过完成修复订单获得金币，并将金币用于购买材料、支付每日杂费、支付固定房租和进行小店升级。玩家需要在“当前生存”和“长期改善”之间取舍。

### 能量

能量代表主角当天的工作状态。接待顾客、修补物品、处理差评或接受额外要求都会消耗能量。能量用于控制单日行动数量，避免玩家只追求收益最大化；加班或接取高消耗订单可能影响第二天状态。

### 真实度

真实度代表主角是否保留自己的判断、节奏和边界。迎合顾客、接受压价、免费修改或改变店铺风格可能降低真实度；保留痕迹、拒绝不合理要求、坚持自己的修复方式可能维持或提升真实度。真实度会影响夜晚日记、主角状态和结局方向。

### 材料

材料是完成修复订单的基础消耗资源。玩家通过夜晚购买或其他渠道补充材料，再在白天订单中消耗。材料系统限制玩家不能无限接单，并让订单选择与库存状态产生关联。

### 口碑

口碑代表小店在顾客中的评价。完成订单、满足顾客需求或处理售后反馈会影响口碑；口碑会改变后续订单和顾客类型，让玩家感受到经营风格正在被顾客记住。

## 5. 主要机制

### 每日经营机制

每日经营以需求栏和订单为主线。玩家先查看当天订单，再选择是否接单。每个订单会消耗材料、时间和能量，并在交付后带来收入、反馈和可能的口碑变化。夜晚通过手机查看反馈、购买材料、结算后进入下一天。

### 订单选择机制

订单由以下要素构成：

- 物品类型
- 损坏程度
- 所需材料
- 消耗时间
- 消耗能量
- 报酬
- 顾客备注

订单按难度可分为简单、普通和困难。订单选择应该形成清晰取舍：高报酬订单通常更消耗材料和能量；低报酬订单可能更温柔、更容易积累好评；特殊订单可能影响真实度或口碑方向。

### 修补机制

修补方式分为三类：

- **完美复原**：收入高、好评上升、能量消耗大。
- **保留痕迹**：收入一般、能量消耗一般，更符合小店温度。
- **创意再造**：收入不稳定、能量消耗中上，可能吸引改造型顾客。

修补结果应同时影响收入、顾客反馈、口碑和真实度。

### 顾客机制

顾客可先按三类设计：

- **高消费且挑剔型顾客**：愿意支付更多，但对结果更苛刻。
- **温柔型顾客**：更容易接受痕迹与不完美，反馈相对稳定。
- **改造型顾客**：期待创意变化，可能带来更高波动。

不同顾客类型会影响订单要求、反馈口径和后续口碑。

### 口碑机制

口碑描述外界如何理解这间小店：

- **完美复原**：容易吸引高消费且挑剔型顾客。
- **保留痕迹**：容易吸引温柔型顾客。
- **创意再造**：容易吸引改造型顾客。

口碑不是单纯好坏，而是影响订单池和顾客组成的方向系统。

### 材料库存机制

材料是修补行为的基础资源。初始材料类型包括：

- 胶水
- 布料
- 木料
- 旧零件
- 颜料

不同修补方式对应不同材料消耗。材料不足时，玩家不能完成对应修补，或只能选择替代方案。

### 店铺升级机制

店铺升级通过收入和材料解锁，主要提升上限或效率：

- **工具架**：每日能量上限少量提升。
- **材料架**：材料库存上限增加。
- **展示角**：口碑提高。
- **休息椅**：恢复能量提升。
- **旧物展示墙**：真实度稍微增加。

升级应服务核心循环，不应在早期压过订单与修补本身。

### 手机机制

手机主要在夜晚使用，承载经营反馈和轻叙事：

- 查看店铺评价
- 查看日记
- 查看朋友圈
- 查看今日收入、材料消耗、能量变化和口碑变化
- 购入新的材料

手机机制适合作为 Alpha / Beta 阶段扩展，MVP 可暂时只保留夜晚结算。

### 边界选择机制

顾客可能提出额外要求，玩家可以接受或拒绝：

- **加急完成**：收入更高，但能量消耗更多。
- **加班**：收入提升，第二天能量降低。
- **免费修改**：口碑极大提升，但收入为 0。
- **压低价格**：口碑提升，收入降低。
- **要求改变店铺风格**：口碑提升，真实度降低。

该系统用于强化主题：小店是否要为了评价和收入持续让步。

## 6. MVP 范围

MVP 只验证最小可玩闭环：

- 查看订单
- 选择订单
- 准备材料
- 修补旧物
- 交付顾客
- 获得收入与反馈
- 夜晚购买材料并结算
- 开启新一天

MVP 暂不实现复杂手机内容、深度口碑分流、复杂店铺升级、多结局和大量剧情。

详见 [MVP 范围说明](MVP.md)。

## 7. 内容与美术方向

- 画面风格应偏温柔、手作、旧物感。
- UI 应优先清楚表达订单、材料、能量和结算变化。
- 美术资产在 MVP 阶段可以使用占位图形，先验证流程。
- 文字语气应生活化、克制、带一点不完美的幽默感。

## 8. 开发路线图

### MVP

实现一轮可玩的日循环，支撑玩家连续经营数天。

### Alpha

扩展订单、顾客、修补方式、口碑与基础升级，形成可重复游玩的结构。

### Beta

加入手机内容、日记、朋友圈、更多反馈文本和数值平衡。

### 内容扩展

补充顾客故事、特殊订单、装饰变化和更长期的小店成长。

## 9. 待补全策划章节

桌面策划稿中，以下章节目前主要是目录骨架，需要后续补成可执行规格：

- **叙事设定**：故事背景、主角设定、店铺设定、顾客群像、叙事结构、事件类型、结局设定。
- **游戏流程**：整体流程、新手引导、订单处理、修补、夜晚结算、阶段推进、结局触发。
- **核心系统**：时间 / 行动点、商品售卖、顾客关系、店铺风格、解锁成长、结算。
- **数值系统**：收入、能量、真实度、材料成本、订单收益、顾客满意度、口碑变化、解锁条件、结局判定。
- **内容表设计**：物品、订单、顾客、事件、材料、数值配置、文本配置和存档数据结构。
- **表现设计**：UI / UX、场景结构、店铺场景、工作台、夜晚房间 / 手机场景、交互点和状态变化。

## 10. 风险与约束

- 系统容易膨胀，早期必须优先保护每日经营闭环。
- “真实度”需要通过具体选择体现，避免变成抽象分数。
- 文本内容可能成为工作量大头，应先建立少量高复用模板。
- 数值压力不能过硬，否则会破坏温柔修补的体验。
- 成长系统不应变成复杂商业扩张，小店应更接近“变稳定、变像自己”，而不是无限做大。
