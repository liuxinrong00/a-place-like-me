# 《不完美的小店》UI 风格方向

## 1. 风格关键词

- 简洁清楚：参考 shadcn/ui 的信息层级、留白、卡片和按钮秩序。
- 柔和系统感：参考 Fluent 的轻量层级、状态反馈和可读性。
- 黄色破旧风：整体像旧纸、旧木、钨丝灯和褪色标签，而不是明亮赛博黄。
- 手作温度：界面应该像小店工作台，不像后台管理系统。

## 2. 当前原型视觉基线

当前程序化 uGUI 原型采用：

- 深褐背景，模拟旧木桌面。
- 泛黄纸张卡片，承载订单、详情和修补方式。
- 暖黄色作为主按钮和选中态。
- 深棕色文字，保证中文可读。
- 少量 muted 色，用于次要说明和资源信息。

对应代码入口：

- `Assets/_Project/Scripts/UI/PrototypeUiTheme.cs`
- `Assets/_Project/Scripts/UI/PrototypeGameController.cs`

## 3. 布局原则

原型阶段从纯三栏工作台升级为“店内平面 + 信息面板”：

1. 左侧大区域：小店平面图，包含收银台、货架、门、公告栏和玩家小人。
2. 独立 UI 场景：默认不加载，和公告栏或床交互后显示订单、详情、修补方式、顾客反馈、夜晚补货。
3. 底部操作区：开始修补、夜晚结算、补货、进入下一天。
4. 玩家可以用 `WASD` / 方向键移动，靠近门或公告栏后按 `E` 互动。

门用于在店铺和卧室之间切换；公告栏用于加载订单 UI 场景；卧室里的床用于加载结算 UI 场景并进入下一天。这样能让经营循环先有“空间感”，再逐步替换成正式场景和美术。

后续正式 UI 可以替换为 Prefab，但“店内空间 + 交互点 + 信息面板”的结构应保持稳定。

## 4. 组件方向

### 卡片

- 用于订单、详情、修补方式、结算摘要。
- 卡片颜色使用旧纸黄，不使用纯白。
- 卡片内部留白要足，避免经营信息挤成表格。

### 按钮

- 主按钮：暖黄色，用于“开始修补”“进入下一天”。
- 次按钮：纸卡色，用于选择订单、选择修补方式、补货材料。
- 选中态：更亮的旧黄，像贴了一张便签。
- 禁用态：低饱和褐色，明确但不刺眼。

### 文本

- 标题：深棕、粗体。
- 正文：深棕。
- 辅助说明：浅褐灰。
- 反馈文本允许更有叙事感，但不要太长，优先一屏可读。

## 5. 色彩基线

- 背景：深褐 / 旧木色。
- 面板：泛黄纸色。
- 次面板：灰黄旧纸色。
- 主色：暖黄 / 旧金色。
- 文字：深棕。
- 正反馈：低饱和绿色。
- 风险/不足：低饱和红棕。

## 6. 不做的事

- 不使用高饱和亮黄。
- 不做复杂动效，先保证流程清楚。
- 不在原型阶段引入复杂 UI 框架。
- 不依赖 TextMeshPro，先用 uGUI `Text` 和系统中文字体保证可运行。

## 7. 下一步 UI 工作

- 将程序化 UI 拆成可维护 Prefab。
- 给订单卡片增加更明确的状态：可做、材料不足、能量不足、已完成。
- 给夜晚结算做单独面板。
- 给门、公告栏、货架、收银台增加更明确的靠近高亮。
- 给手机、日记和朋友圈预留入口，但不急着实现完整系统。
